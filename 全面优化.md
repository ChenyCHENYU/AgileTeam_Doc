## 二、优化方案

### 2.1 性能优化（P0 — 优先执行）

#### 2.1.1 移除 Fabric.js 死依赖

`package.json` 中 `fabric` 在 `dependencies` 但项目中 **没有任何 import/require**（签名文档已通过 DemoIframe 嵌入 Robot Admin 演示）。

**操作**：

```bash
bun remove fabric
```

**收益**：减少安装体积，消除无用依赖的安全审计噪音。

#### 2.1.2 评论组件懒加载

当前 Giscus 评论组件在所有文档页面底部加载，产生额外的 iframe 请求。

**优化方案**：

```vue
<!-- 改为 IntersectionObserver 触发加载 -->
<template>
  <div ref="commentRef">
    <Giscus v-if="visible" ... />
  </div>
</template>

<script setup>
const commentRef = ref();
const visible = ref(false);

onMounted(() => {
  const observer = new IntersectionObserver(
    ([entry]) => {
      if (entry.isIntersecting) {
        visible.value = true;
        observer.disconnect();
      }
    },
    { rootMargin: "200px" },
  );
  observer.observe(commentRef.value);
});
</script>
```

#### 2.1.3 图片资源优化

- 添加 `loading="lazy"` 属性到 ImgWrap 组件
- 新增图片统一使用 WebP 格式

### 2.2 可维护性优化（P1 — 重点推进）

#### 2.2.1 CSS 架构拆分

将 `custom.css` 按职责拆分，降低单文件复杂度：

```
theme/styles/
├── variables.css         # CSS 变量（:root 和 .dark）
├── layout-desktop.css    # >= 1440px 宽屏布局增强
├── layout-ultrawide.css  # >= 1920px 超宽屏优化
├── navbar.css            # 导航栏修复
├── content.css           # 文档内容排版
├── home.css              # 首页样式（已存在）
├── newBadge.css          # NEW 标记（已存在）
└── index.css             # 统一入口 @import
```

**主题入口改为**：

```javascript
import "./styles/index.css";
```

#### 2.2.2 Sidebar 配置模块化

将 `sidebarArr.js` 按文档类型拆分：

```
_config/sidebar/
├── index.js            # 聚合导出
├── robot-guide.js      # Robot Admin 指南
├── robot-components.js # 组件文档
├── robot-plugin.js     # 插件文档
├── web.js              # 前端文档
├── po.js               # 产品文档
├── ui.js               # 设计文档
├── rear-end.js         # 后端文档
├── qc.js               # 测试文档
└── manage.js           # 管理文档
```

#### 2.2.3 开启死链检测

```javascript
// config.js
// 移除 ignoreDeadLinks: true
// 改为精准忽略已知外部链接
ignoreDeadLinks: [
  /^https:\/\/robotadmin\.cn/,  // 演示站点
],
```

**配套操作**：

- 构建时检查所有死链并修复
- CI 中添加链接检查步骤

## 六、风险与注意事项

1. **CSS 拆分风险**：样式优先级可能因文件导入顺序变化，需逐步验证
2. **评论系统**：Giscus 懒加载需验证与 VitePress SPA 路由的交互
3. **向后兼容**：所有优化不应改变文档站的公共 URL 结构
4. **VitePress 版本**：当前 2.0.0-alpha.9 为官方 2.x 分支最新版本，跟进官方更新即可，无需回退 1.x
